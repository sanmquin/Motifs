# 7. Matrix Embeddings Documentation (22-Dimensional Base Connectivity Embedding)

This document provides a comprehensive specification and semantic breakdown of the **22-dimensional connected component-based matrix embedding** exported in `7.matrix_embeddings.csv`.

---

## 1. Executive Summary & Design Rationale

In the Abstraction and Reasoning Corpus (ARC-AGI-2), grids are fundamentally structured around discrete visual objects rather than raw, unorganized pixels. To represent individual matrices (including input and output states across both training and evaluation tasks) with semantic and topological awareness, we project each grid into a **22-dimensional core connectivity embedding space**.

Rather than utilizing complex, dataset-dependent, or over-parameterized neural descriptors (which are prone to overfitting on sparse datasets), this embedding space relies on rigorous, mathematically defined topological descriptors. By segmenting components under different connectivity configurations and adjacency definitions, we capture structural invariance, object counting, sizing, scaling, and color patterns.

---

## 2. Feature Segmentation & Connectivity Taxonomy

The embedding vector is built by extracting connected components under **four distinct configurations** resulting from the Cartesian product of two connectivity standards and two grouping rules:

### A. Adjacency Connectivity Standards
1.  **4-Connectivity (Orthogonal)**: Two pixels of the target configuration are considered part of the same component if and only if they share a horizontal or vertical edge.
2.  **8-Connectivity (Orthogonal + Diagonal)**: Two pixels are considered part of the same component if they share a horizontal, vertical, or diagonal edge/corner.

### B. Grouping & Segmentation Rules
1.  **Same-Color**: Pixels are grouped together if they share the *same uniform non-background color*. This captures individual uniform monochromatic elements.
2.  **Non-Background (NonBG)**: All active foreground pixels (color $\neq 0$) are segmented together regardless of their color. This captures unified multicolored foreground structures and multi-colored composite objects.

*Note: Following the canonical corpus definition, background pixels (color 0) are excluded from being segmented as components.*

---

## 3. Detailed Semantic Feature Dictionary

The 22 features are organized into five distinct thematic groups:

### Group A: Component Counts (4 Features)
These features quantify the cardinality of visual components across the four configuration spaces, acting as diagnostic counts for object-preservation, division, or merging rules.

1.  **`4_SameColor_count`**: The total count of 4-connected, single-color monochromatic components in the grid.
2.  **`4_NonBG_count`**: The total count of 4-connected, multi-colored foreground components.
3.  **`8_SameColor_count`**: The total count of 8-connected, single-color monochromatic components.
4.  **`8_NonBG_count`**: The total count of 8-connected, multi-colored foreground components.

### Group B: Component Sizes (8 Features)
These features measure the spatial extent (number of pixels) of the components, capturing scaling, resizing, or average component mass.

5.  **`4_SameColor_mean_size`**: The average pixel area of 4-connected monochromatic components.
6.  **`4_SameColor_max_size`**: The pixel area of the largest 4-connected monochromatic component.
7.  **`4_NonBG_mean_size`**: The average pixel area of 4-connected multi-colored foreground components.
8.  **`4_NonBG_max_size`**: The pixel area of the largest 4-connected multi-colored foreground component.
9.  **`8_SameColor_mean_size`**: The average pixel area of 8-connected monochromatic components.
10. **`8_SameColor_max_size`**: The pixel area of the largest 8-connected monochromatic component.
11. **`8_NonBG_mean_size`**: The average pixel area of 8-connected multi-colored foreground components.
12. **`8_NonBG_max_size`**: The pixel area of the largest 8-connected multi-colored foreground component.

### Group C: Component Elongation & Shape (8 Features)
These features quantify the geometric aspect ratio/shape of the components, modeling elongation and helping distinguish square-like structures from line-like or elongated objects.
*For each component, elongation is calculated as:*
$$\text{Elongation} = \frac{\max(H_{\text{box}}, W_{\text{box}})}{\min(H_{\text{box}}, W_{\text{box}})}$$
*Where $H_{\text{box}}$ and $W_{\text{box}}$ are the height and width of the component's minimal bounding box. If no components exist, the elongation defaults to $1.0$.*

13. **`4_SameColor_mean_elong`**: The average elongation across 4-connected monochromatic components.
14. **`4_SameColor_max_elong`**: The maximum elongation across 4-connected monochromatic components.
15. **`4_NonBG_mean_elong`**: The average elongation across 4-connected multi-colored foreground components.
16. **`4_NonBG_max_elong`**: The maximum elongation across 4-connected multi-colored foreground components.
17. **`8_SameColor_mean_elong`**: The average elongation across 8-connected monochromatic components.
18. **`8_SameColor_max_elong`**: The maximum elongation across 8-connected monochromatic components.
19. **`8_NonBG_mean_elong`**: The average elongation across 8-connected multi-colored foreground components.
20. **`8_NonBG_max_elong`**: The maximum elongation across 8-connected multi-colored foreground components.

### Group D: Auxiliary Global Topological Metrics (2 Features)
These features track color palletization diversity and grid margin boundary conditions, capturing peripheral containment.

21. **`color_diversity`**: The number of unique active (non-background) colors present across the 8-connected monochromatic components.
22. **`border_touching_ratio`**: The proportion of 8-connected monochromatic components whose bounding boxes touch at least one of the four grid borders (top, bottom, left, or right).

## 4. Export File Schema (`7.matrix_embeddings.csv`)

The exported file is structured as a comma-separated value (CSV) file containing exactly **27 columns** (5 metadata columns plus the 22 feature dimensions):

| Column Name | Data Type | Description |
| :--- | :--- | :--- |
| **`task_id`** | String (Hex) | The unique identifier of the ARC task/puzzle (e.g., `00576224`). |
| **`subset`** | Category | The dataset split: `Training` (1,000 tasks) or `Evaluation` (120 tasks). |
| **`pair_type`** | Category | The pair category: `train` (demonstration pairs) or `test` (test pairs). |
| **`pair_idx`** | Integer | The index of the pair within the task (0-indexed). |
| **`grid_type`** | Category | Indicates whether the grid is the `input` or `output` grid of the pair. |
| **`[Features 1-22]`** | Float | The 22 connectivity features detailed in Section 3 of this document. |

---

## 5. Summary Statistics & Robustness Checks

During validation across the ARC-AGI-2 training set (8,616 matrices) and evaluation set (1,048 matrices):
*   **Completeness**: $100\%$ of matrices are fully processed without any NaN or infinite values.
*   **Scale Invariance**: All mean and max size features are expressed in pixels, but can be normalized relative to the grid area ($H \times W$) for scale-free context.
*   **Size Constraints**: Monochromatic and foreground components with a pixel area of strictly less than 4 pixels (i.e., small noise or solitary pixels) are excluded during feature computation to filter out high-frequency visual noise, ensuring focus on primary structural motifs.
