# 7. Joint Input-Output Pair Embeddings Documentation (221-Dimensional Representation)

This document provides a comprehensive specification and semantic breakdown of the **221-dimensional joint transformation embedding** exported in `7.io_pair_embeddings.csv`.

---

## 1. Executive Summary & Design Rationale

To capture, represent, and solve visual rules in the Abstraction and Reasoning Corpus (ARC-AGI-2), encoding isolated grids is insufficient. The underlying logic of a puzzle is defined by how the input grid transforms into the output grid. Our joint Input-Output (I/O) pair representation captures this "transformation signature".

Each I/O pair (including train and test pairs across training and evaluation tasks) is projected into a **221-dimensional joint embedding space** composed of:
1.  **Input Matrix Core Features (22 dimensions)**: The 22-dimensional base connectivity features of the **input matrix** alone, establishing the starting topological context.
2.  **Joint Transformation Features (199 dimensions)**: A multi-dimensional feature set detailing geometric resizing, component counting deltas, Shannon complexity updates, centroid translations, bipartite color mappings, and component taxonomy shifts.

---

## 2. Combined Feature Taxonomy & Specification

The 221 features are organized into the following nine distinct groups:

### Group A: Input Core Context (22 Features)
These are the 22 core connected component connectivity features computed exclusively on the **input** matrix. They are prefixed with `in_22_` to distinguish them from joint transformation features. For a complete definition of these 22 features, please consult `7.Matrix_Embeddings_Documentation.md`.
*   **`in_22_4_SameColor_[count/mean_size/max_size/mean_elong/max_elong]`** (5 features)
*   **`in_22_4_NonBG_[count/mean_size/max_size/mean_elong/max_elong]`** (5 features)
*   **`in_22_8_SameColor_[count/mean_size/max_size/mean_elong/max_elong]`** (5 features)
*   **`in_22_8_NonBG_[count/mean_size/max_size/mean_elong/max_elong]`** (5 features)
*   **`in_22_color_diversity`** (1 feature)
*   **`in_22_border_touching_ratio`** (1 feature)

---

### Joint Transformation Features (199 Features)

The remaining 199 features are joint metrics modeling the step-by-step structural transformations between input ($I$) and output ($O$) grids.

### Group B: Individual Grid Properties (2 * 36 = 72 Features)
Features prefixed with `in_` and `out_` capture separate baseline properties for input and output grids (36 features each):
*   **Grid Size & Aspects**: `height`, `width`, `area`, `aspect_ratio`, `is_square`.
*   **Color Distributions**: `color_ratio_0` to `color_ratio_9` (10 features mapping the relative pixel density of each color, including background 0).
*   **Active Pixels**: `active_ratio` (percentage of active pixels), `entropy` (Shannon entropy of pixel values).
*   **Symmetries**: `sym_h` (horizontal), `sym_v` (vertical), `sym_rot90` (90° rotation), `sym_rot180` (180° rotation), `sym_rot270` (270° rotation).
*   **Margins**: `border_active_ratio` (percentage of active pixels along margins), `corners_active_ratio` (active corners).
*   **Connectivity**: `cc_same_col_4_count`, `cc_same_col_4_mean_size`, `cc_same_col_4_max_size`, `cc_same_col_4_std_size`, `cc_nb_8_count`, `cc_nb_8_mean_size`, `cc_nb_8_max_size`, `cc_nb_8_std_size`, `nested_cc_count`.
*   **Component Shapes**: `cc_type_count_lines`, `cc_type_count_squares`, `cc_type_count_rectangles`, `cc_type_count_junctions`, `cc_type_count_diagonals`, `cc_type_count_others`.

---

### Group C: Spatial & Grid Geometry Transformations (4 Features)
These model the scaling of grid boundaries, capturing whether a puzzle involves grid enlargement, cropping, or aspect ratio changes.
*   **`resize_h_ratio`**: Height ratio ($\frac{H_{out}}{H_{in}}$).
*   **`resize_w_ratio`**: Width ratio ($\frac{W_{out}}{W_{in}}$).
*   **`resize_area_ratio`**: Area scaling factor ($\frac{\text{Area}_{out}}{\text{Area}_{in}}$).
*   **`resize_aspect_ratio_diff`**: Change in aspect ratio ($\text{Aspect}_{out} - \text{Aspect}_{in}$).

---

### Group D: Connected Component Cardinality Shifts (3 Features)
These track changes in component and nested structure population, signaling splitting or merging.
*   **`cc_same_col_4_count_diff`**: Delta in 4-connected same-color components ($\text{Count}_{out} - \text{Count}_{in}$).
*   **`cc_nb_8_count_diff`**: Delta in 8-connected non-background components.
*   **`cc_nested_count_diff`**: Delta in nested bounding boxes.

---

### Group E: Background Shifts and Color Preservation (4 Features)
These measure global color mapping profiles, background alterations, and additions or removals.
*   **`bg_changed`**: Binary flag indicating whether the top-left corner cell changed color between input and output (approximate background shift).
*   **`num_colors_added`**: Count of unique active colors present in the output grid but absent in the input grid.
*   **`num_colors_removed`**: Count of unique active colors present in the input grid but completely deleted in the output grid.
*   **`color_preservation_ratio`**: Intersection-over-union (IoU) of active input and output colors.

---

### Group F: Overlap Bipartite Color Mapping (100 Features)
This builds a $10 \times 10$ bipartite color transformation matrix, measuring what fraction of pixels of color $C_{\text{in}}$ in the input overlap with color $C_{\text{out}}$ in the output. This captures precise same-color substitutions, partial color mappings, and solid color fills:
*   **`color_map_{C_in}_to_{C_out}`** (where $C_{\text{in}}, C_{\text{out}} \in [0, 9]$): Normalized overlap mapping frequency.

---

### Group G: Visual Order & Complexity Shifts (2 Features)
These track changes in global pixel disorder and density.
*   **`active_ratio_diff`**: Change in active pixel ratio ($\text{ActiveRatio}_{out} - \text{ActiveRatio}_{in}$).
*   **`entropy_diff`**: Change in Shannon entropy ($\text{Entropy}_{out} - \text{Entropy}_{in}$).

---

### Group H: Centroid Translation Offsets (2 Features)
These measure the directional shift of the global center-of-mass of active pixels, capturing translation invariance.
*   **`centroid_shift_y`**: Shift of active pixel centroid along the vertical axis ($\text{Cy}_{out} - \text{Cy}_{in}$).
*   **`centroid_shift_x`**: Shift of active pixel centroid along the horizontal axis ($\text{Cx}_{out} - \text{Cx}_{in}$).

---

### Group I: Component Shape Taxonomy Shifts (6 Features)
These capture deltas in the populations of standard geometric motifs:
*   **`cc_type_diff_lines`**: Delta in count of straight 1D same-color orthogonal lines (length $\ge 3$).
*   **`cc_type_diff_squares`**: Delta in solid same-color squares (size $\ge 2\times 2$).
*   **`cc_type_diff_rectangles`**: Delta in solid same-color rectangles (non-square, size $\ge 2\times 2$).
*   **`cc_type_diff_junctions`**: Delta in orthogonal same-color junctions (corners, L-shapes, T-shapes, crosses of size $\ge 3$).
*   **`cc_type_diff_diagonals`**: Delta in diagonal same-color structures.
*   **`cc_type_diff_others`**: Delta in unclassified irregular same-color components.

---

## 3. Export File Schema (`7.io_pair_embeddings.csv`)

The exported file is structured as a comma-separated value (CSV) file containing exactly **225 columns** (4 metadata columns plus the 221 feature dimensions):

| Column Name | Data Type | Description |
| :--- | :--- | :--- |
| **`task_id`** | String (Hex) | The unique identifier of the ARC task/puzzle (e.g., `00576224`). |
| **`subset`** | Category | The dataset split: `Training` (1,000 tasks) or `Evaluation` (120 tasks). |
| **`pair_type`** | Category | The pair category: `train` (demonstration pairs) or `test` (test pairs). |
| **`pair_idx`** | Integer | The index of the pair within the task (0-indexed). |
| **`[Features 1-221]`** | Float | The 221 features detailed in Section 2 of this document. |
